# Blog Migration - blog.rovonnrussell.com -> rovonnrussell.com/blog

The blog moves from a subdomain to a subdirectory. This consolidates SEO into the main domain and simplifies the maintenance footprint to one site.

## What's moving

- The blog itself: from `blog.rovonnrussell.com/blog/<slug>` (and `blog.rovonnrussell.com/<slug>` if any) to `rovonnrussell.com/blog/<slug>`
- The visual treatment: matched to the main site (cream/charcoal/orange palette, Playfair + DM Sans)
- The voice: recalibrated against 21 real LinkedIn posts (see `docs/VOICE_REFERENCE.md`)
- The IA: nav now points at `/blog` instead of the subdomain

## What's NOT moving

- The 15 AI Agents post (was AI-written, not migrated). Drop a 410 Gone or 301 to the new blog index for that specific URL.
- Any draft posts that haven't been validated against the new voice doc.

## Required hosting changes

This site is deployed via Lovable. The redirects below need to be configured at the hosting layer (Lovable, Cloudflare, or whatever DNS / edge proxy is in front of the subdomain).

### 1. Subdomain-level 301 redirects

For every URL on `blog.rovonnrussell.com`, redirect to the matching subdirectory URL on `rovonnrussell.com/blog`. The recommended setup:

| Source | Destination | Status |
|---|---|---|
| `blog.rovonnrussell.com` | `rovonnrussell.com/blog` | 301 |
| `blog.rovonnrussell.com/blog` | `rovonnrussell.com/blog` | 301 |
| `blog.rovonnrussell.com/blog/lessons-from-building-15-ai-agents` | `rovonnrussell.com/blog` | 301 (or 410 Gone) - this post is retired |
| `blog.rovonnrussell.com/blog/<any-other-slug>` | `rovonnrussell.com/blog/<slug>` | 301 |
| `blog.rovonnrussell.com/<any-path>` | `rovonnrussell.com/<path>` | 301 (catch-all) |

If the subdomain runs through Cloudflare, the Page Rule pattern is:
```
blog.rovonnrussell.com/blog/*   ->   https://rovonnrussell.com/blog/$1   (301)
blog.rovonnrussell.com/*        ->   https://rovonnrussell.com/$1        (301)
```

### 2. App-level redirects (already shipped)

`public/_redirects` contains:
```
/blogs/*    /blog/:splat   301
/*    /index.html   200
```

This handles the misspelling `/blogs` -> `/blog` at the hosting layer (Netlify, Vercel, and Lovable's hosting all read this file).

The React Router redirects in `src/App.tsx` (`/blogs` -> `/blog`, `/blogs/:slug` -> `/blog/:slug`) are a backup for client-side navigation.

### 3. Sitemap and robots updates

After the migration:
- Submit the new sitemap (`https://rovonnrussell.com/sitemap.xml`) to Google Search Console
- Submit the new RSS feed (`https://rovonnrussell.com/rss.xml`) to feed readers
- Update any external backlinks pointing at `blog.rovonnrussell.com` where you control them (LinkedIn, X bio, email signatures)
- In Search Console, mark the subdomain as a property and use the Change of Address tool to signal the migration

## Validation checklist

After deploying:

- [ ] `rovonnrussell.com/blog` loads and shows the new index
- [ ] `rovonnrussell.com/blog/why-i-run-three-brands-instead-of-one` loads (once `published: true` is set on the sample post)
- [ ] `rovonnrussell.com/blog/category/building-in-public` loads
- [ ] `rovonnrussell.com/rss.xml` is generated by `npm run build` and serves correctly
- [ ] Old subdomain URLs return 301 to the matching new URL (test with `curl -I`)
- [ ] Search Console shows the change-of-address request submitted
- [ ] Internal links across the site (Navbar, Footer, Index articles section) point at `/blog`
- [ ] The `Articles` link in the Navbar has been replaced with `Blog`

## What happens if the redirects aren't perfect

Worst case: some link equity is lost during transition. The voice doc, the editorial calendar, and the new on-page SEO will rebuild authority quickly. The cleanest move is to handle the redirects properly on day one, but a partial setup won't sink the migration.
